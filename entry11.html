Eleventh Entry

Jason Lan, June 24, 2015. 12:00

Overtime

Yesterday I stayed and worked overtime with the rest of the group. People at the company often work overtime whenever there is work that needs to be completed. I was able to be slightly more involved in the work, even creating part of the code that was added to the websites. The company has 8 websites for different parts of the world that they sell to, each website customized for the target audience in language, style, and format. I used Macromedia Fireworks (definitely torrented 2003 version of Adobe Fireworks) to create hotspots with URL links on a picture, which was much simpler than using Photoshop to slice parts of the image from the background and attach links to them. I then sent the code to ZhangRu so she could make a version for every website and upload it onto the website server. The website server is secured with individual IDs and Passwords so that it is clear who made the changes to the website.

The whole process took about two hours to ensure that all the links and images were functional. Honestly I feel like I could have done everything in an hour, but I suppose it's easier to watch than to actually pick out all the errors. During this time, the semi-functional code was uploaded directly onto the webpages to test, so customers who happened to be on the webpages they were editing could see the code. The added content was for a new sale that the website was hosting, and was hidden after the editing and proofreading process. I remember when I used Dreamweaver to make changes to the Highly Gifted Magnet website in middle school that I made sure everything worked locally before uploading it online, and only checked online as a final check. The reason the process is so slow is that each website needs to be checked separately and there is only one local check before uploading the code online. Any additional edits to match the different websites are checked on the website server instead of a local server. The website has too many files to be on a local server, but I feel that there should be an additional test server containing an exact copy of all the files on the real website (maybe there is) so that code can be tested on that server before final publishing. This would also serve as a backup in case website files are lost or corrupted.

Gains from Overtime

I learned a couple things during the overtime experience. Firstly, overtime is very tiring. I felt like falling asleep and it was only 9:30 when I left. My coworkers told me that there were some overtime runs that lasted until midnight or even beyond that. The next morning was painful for me as well. Secondly, I do have a lot of experience with computers from my personal use or website managing. I remembered the skills from using Adobe Dreamweaver even though I haven't used it since middle school and didn't even remember that I had that knowledge until it became useful. Thirdly, I don't think I'm quite ready for full time work yet. Although I am learning a lot I don't think I can manage to keep up this schedule for a year since I already feel that I am lacking exercise and need more breaks in between computer work. Overtime is even more out of the question. Lastly, however, I feel like I am capable of doing company work. Things like the overtime project are so second nature to me that I could do it on my own after watching someone do only a quarter of it. I don't even need someone to walk me through or provide an example if it's something so simple. I could easily have done the rest as soon as I watched someone insert the code online. More advanced coding is still beyond me but the company still needs simple, time-consuming work to be done and I have learned enough to do that, without losing in speed or precision to more experienced workers. In terms of computer usage, I'm sure I don't fall short on experience in comparison to the other workers, at least not by far.
