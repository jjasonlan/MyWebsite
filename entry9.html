Second Week of Interning

This week I broadened my studying and did a lot of experimentation with npm and node.js, which are relatively newer computing tools. npm allows programmers to download packages from the internet through the command prompt, using node.js, a client-side, event driven platform. Sharing code and files is extremely important for creating new progress in programming, and npm makes that process easy and accessible so people can publish their packets and install packets from others. Through my research I obtained a much better understanding of what part my company group plays in the larger scale of the company and how other groups are all connected to making the company website functioning and updated.

My group is in charge of front end operations, also known as client side scripting. This has heavy reliance on Javascript and its corresponding programs (like node.js!). Strictly speaking, the client is the browser where a program is run, or the company website is viewed. Using Javascript, as well as basic HTML and CSS, the client side scripts run on the computer after a web page is loaded. So, the front end is in charge of the immediate responses to user interaction, as well as a large portion of what is seen on a webpage.

Another group in the company is in charge of back end operations, or server side scripting. Server side scripting works with data and information sent to the webpage before the page is loaded. Server side scripts are not seen in code, but as HTML as the webpage is retrieved from the server via internet. Back end operations frequently use SQL and PHP, as well as other server side scripts.

Because my group works very little or not at all with SQL and PHP scripts, I decided to stop learning them. I think front end operations are easier for me to understand since it is more difficult to obtain server data than to create my own webpages as an intern. LaoZheng, my boss, pointed me to Grunt and Gulp, which are task based command line tools, somewhat similar to npm. Grunt was the standard for a long time, and is still used by many websites. The newest upgrade to front end operations is Gulp, which is still developing. Gulp has various differences from Grunt, which make it more appealing for future acceptance in the programming community. Grunt failed in its reliance on global installations with can cause chaos among different projects. Gulp does not need to installed globally to be used. Gulp shines because it is a streaming build tool, which means it can add more plugins while only reading and writing to disk once.
